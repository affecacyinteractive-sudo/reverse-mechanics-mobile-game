## KEYSTONE-BUILD — Final (LLM-only, chunk-first, no rigid JSON)

### Purpose

Create a **school-typed keystone**: the canonical, publishable “what this meant” artifact for a run. It is written for **human recall and replay**, not for machine execution.

---

## Input Grammar (UI layer)

Build a keystone from **[Run Output : Targets]** in the genre of **[Keystone School : Intent]**.
If present, reuse canon wording from **[Project Vocabulary : Chronicle]** and ground naming with **[Proof Excerpt : Packets]**.

Required blanks: **Run Output**, **Keystone School**.
Optional blanks: **Project Vocabulary**, **Proof Excerpt**.

---

## Output Contract (chunk-first, hard)

Output a **Keystone Pack** as chunks with the following exact titles:

Required:

* **Keystone Title** (TEXT) — short, specific, school-flavored; not generic (“Update”, “Refactor”).
* **Keystone Body** (TEXT) — a compact, card-sized summary written in the school’s genre.

Optional (include only when it genuinely improves reuse):

* **Anchors** (TEXT) — a few handle-like terms, one per line, chosen from terms present in the inputs.
* **Code Staple** (CODE) — a tiny excerpt only if it materially improves concreteness; must match provided evidence.

Hard rules:

* No patch plans, no implementation steps, no chunk dumps, no transcripts.
* No “next move” suggestions, no action references.
* Do not invent file paths, identifiers, or behaviors not supported by Run Output / Proof Excerpt.
* If inputs are thin, choose the smallest reasonable summary and include uncertainty as `Assumed:` lines at the end of Keystone Body (not as questions).

School-genre requirements for Keystone Body:

* **FI** — what is now possible, where it plugs in, and a sense of closure/proof.
* **FU** — what is observably true, including a boundary/cliff or coupling insight when present.
* **FPR** — how to understand/navigate/name what exists, emphasizing legibility.
* **FA** — the mechanism name, governing rule(s), invariant(s), and degrees of freedom.
* **FS** — what got instantiated/stitched/extended, with a seam guarantee or proof point.
* **FP** — corridor decisions: north star / done receipt / scope fence / tripwires.

---

## Recommended Chunk Beat Guidance (for writing quality, not required output)

Keystone Title should read like a commit message for meaning (“Saved filters now persist across sessions”), not like a category label. Keystone Body should read as a stable summary someone can open a week later and immediately understand what changed and why it matters, without needing the full run transcript.

---
