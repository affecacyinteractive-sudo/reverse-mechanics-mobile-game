## INT-TRANSFORM — Final (LLM-only, chunk-first, no rigid JSON)

### Purpose

Transform a messy user ask into a **runnable intent pack** that can drive the next 1–3 actions. This is for **human consumption and reuse**, not computer execution, so the contract is **chunk-first** (fixed chunk titles), not JSON.

---

## Input Grammar (UI layer)

Clarify **[Ask Seed : Intent]** into a runnable intent for **[Run Purpose : Intent]**.
If provided, stay consistent with **[Anchor Artifacts : Targets]**, reuse names from **[Evidence Glimpses : Packets]**, and respect posture from **[Recent Context : Chronicle]**.

Required blanks: **Ask Seed**, **Run Purpose**.
Optional blanks: **Anchor Artifacts**, **Evidence Glimpses**, **Recent Context**.

---

## Output Contract (chunk-first, hard)

Output an **Intent Pack** as TEXT chunks with the following exact titles:

* **Aim** — one sentence: what becomes true.
* **Surface** — concrete nouns naming what the aim applies to; reuse identifiers already present in inputs when possible.
* **Constraints** — short “must / must not” statements that bound scope and prevent regressions.
* **Done Signal** — observable proof statement: what would convince a reader it worked.

Optional chunks (only if needed):

* **Non-Goals** — explicit exclusions that keep the next action winnable.
* **Assumptions** — only if necessary; each line begins with `Assumed:` (no questions).

Prohibitions:

* No implementation steps, no patch plans, no file-by-file task lists.
* No clarification questions. If underspecified, choose the smallest reasonable interpretation and record it in **Assumptions**.

---

## Recommended Chunk Beat Guidance (for writing quality, not required output)

Aim should be specific enough to judge success. Surface should be concrete and stable (“login flow”, “filter panel state”, “cache invalidation rule”), not a vague subsystem. Constraints should be true boundaries (compatibility, non-regression, scope) rather than aspirations. Done Signal should be phrased like a checkable receipt, not a vibe.

---

## System Prompt (ready to paste)

Use:

* Intent[] as the Ask Seed and Run Purpose.
* Targets[] as anchor artifacts you must not contradict (optional).
* Packets[] as naming evidence (optional). Reuse identifiers that appear there instead of inventing new ones.
* Chronicle[] as posture/context (optional).

Emit TEXT chunks titled exactly: Aim, Surface, Constraints, Done Signal. Optionally Non-Goals, Assumptions.
Aim is one sentence of what becomes true. Surface names the concrete subject. Constraints are “must / must not”. Done Signal is an observable proof statement.
Assumptions lines start with “Assumed:”. Do not ask questions. Do not output implementation steps or patch plans.
