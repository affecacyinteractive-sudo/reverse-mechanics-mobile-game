## INT-03 — Intent Generator (Options)

### Input Grammar

**Simple prose (2 required blanks)**
Generate 2–4 intent options from **[Raw Ask : Intent]** for **[Intended Use : Intent]**.

**Required blanks (2)**

* **[Raw Ask : Intent]** — the user’s messy message (what they typed).
* **[Intended Use : Intent]** — what the chosen option will drive next (e.g., “FS-06 extend”, “FI-02 fix”, “FP-10 scope fence”).

**Pro prose (optional boosters)**
Generate 2–4 intent options from **[Raw Ask : Intent]** for **[Intended Use : Intent]**, anchored by **[Existing Anchors : Targets]** and shaped by **[Reality Notes : Chronicle]**.

**Optional blanks (descriptions)**

* **[Existing Anchors : Targets]** — prior artifact packs that should constrain or anchor the options (surface/contract/plan already chosen).
* **[Reality Notes : Chronicle]** — constraints, observed behavior, known risks that should influence which options are plausible.

---

### What to Produce

Produce an **Intent Options Pack**: **2–4 distinct Intent Pack variants**, each runnable for **1–3 downstream actions**.

Rules for each variant:

1. **Aim** — one sentence outcome.
2. **Subject / Surface** — concrete nouns for what it’s about.
3. **Constraints** — 1–3 non-negotiables that define the option.
4. **Done Signal** — observable success statement.
5. **Non-Goals (optional)** — what this option explicitly excludes.
6. **Assumptions (optional)** — only if needed; write as **“Assumed: …”**.

Rules for the option set as a whole:

* Variants must be **meaningfully different** (scope, posture, constraints, or success definition), not rewordings.
* Do **not** include implementation steps, file plans, or patch plans.
* Include a **Pick Guidance** chunk that states the **selection heuristic** in plain language (“Pick A when…, pick B when…”).

---

### Recommended Chunk Beat Categories (with 1-line descriptions)

* **Option Set Frame** — what kind of options these are and what stays common across them.
* **Variant A (Intent Pack)** — a complete, runnable intent with its own constraints + done signal.
* **Variant B (Intent Pack)** — a second runnable intent that differs in scope/posture/constraints.
* **Variant C / D (optional)** — additional variants only if they add real strategic difference.
* **Key Differences** — the single axis that differentiates the variants (scope vs rigor vs risk vs speed).
* **Pick Guidance** — simple “choose this when…” rules so a user can commit to one option.
