INPUT GRAMMAR *(Pattern Lineage / where it sits)*
### Simple prose (2 required INPUT blanks)
Place **[Primary Pattern : Targets]** on a map for **[Placement Intent : Intent]**.

**Required INPUT blanks (2)**
- **[Primary Pattern : Targets]** — the pattern you’re placing (the one you want to make choosable).
- **[Placement Intent : Intent]** — why you’re placing it (choose correctly, compare options, teach the pattern family).

### Pro prose (same core + action-specific optional INPUT blanks)
Place **[Primary Pattern : Targets]** on a map for **[Placement Intent : Intent]**,
using **[Sibling Evidence : Packets]** to compare concrete seams/shapes and **[Tradeoff Notes : Chronicle]** to capture how each option behaves in practice.

**Optional INPUT blanks (descriptions)**
- **[Sibling Evidence : Packets]** — snippets that show how this pattern and its near-siblings manifest (seams, shapes, costs).
- **[Tradeoff Notes : Chronicle]** — observed wins/losses, confusions, “this works when…” and “this fails when…” notes.

WHAT TO PRODUCE (contract)
Produce ONE Pattern Lineage placement that makes the pattern choosable:
- **Parent Frame:** 1 sentence describing the broader family it belongs to.
- **Sibling Alternatives:** 2–3 nearby patterns that solve the same problem differently.
- **Tradeoff Axis:** the single dimension that separates the primary from siblings.
- **Contrast Note (optional):** one sentence sharpening the distinction.
- **Selection Heuristic:** “pick this when…” guidance (1–2 sentences).
- **Misfit Warning:** “avoid when…” guidance (1 sentence).
- **Child Specializations (optional):** 1–2 specializations tied to the same tradeoff axis.
- **Closure Check:** one sentence confirming you picked the right pattern vs siblings.
