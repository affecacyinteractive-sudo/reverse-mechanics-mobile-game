INPUT GRAMMAR *(Pattern Instance / instantiation)*
### Pro prose (same core + action-specific INPUT blanks)
Instantiate the minimal workable instance of **[Source Pattern : Targets]** inside **[Destination Slice : Targets]**, anchored by **[Landing Evidence : Packets]** and **[Fit Notes : Chronicle]**.

**Required INPUT blanks (4)**
- **[Source Pattern : Targets]** — the abstraction/keystone you’re turning into something concrete.
- **[Destination Slice : Targets]** — where the instance will live (the slice/feature/flow receiving it).
- **[Landing Evidence : Packets]** — snippets showing the landing spot, existing seams, call sites, and constraints.
- **[Fit Notes : Chronicle]** — expected behavior, constraints, pitfalls, and why this pattern is being instantiated here.


WHAT TO PRODUCE (contract)
Produce a concrete instance of the pattern inside the destination slice:
1) Pattern Instance Identity: one sentence naming the concrete instance and where it lives.
2) Placement / Seam Location: where the instance attaches (file/module/boundary/handler).
3) Invocation / Usage Seam: how other code calls/consumes it (shape/signature).
4) Pattern → Code Mapping: one tight mapping from abstraction parts to concrete surfaces.
5) Patch Segment(s): 1–3 directional patch segments that wire in the instance (TEXT framing + CODE).
6) Closure Check: one sentence stating the observable proof that the instance works.
